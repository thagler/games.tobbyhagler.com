# App Store Release Checklist

## Pre-Submission

- Confirm build/test status from implementation skills.
- Verify version/build numbers and changelog notes.
- Confirm metadata assets are complete (description, screenshots, categories).

## App Store Connect Setup

- Set pricing and availability.
- Confirm age rating and content declarations.
- Confirm privacy nutrition labels and data-use declarations.

## TestFlight

- Validate internal test group coverage.
- Validate external test plan and release notes.
- Confirm crash and blocker triage before final submission.

## Submission and Rollout

- Choose release method (`manual`, `automatic`, phased).
- Submit version and track review status.
- Record review outcomes and action items in release notes/history.
